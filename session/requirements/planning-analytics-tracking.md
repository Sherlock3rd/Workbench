# 策划案数据打点 Skill 会话

## 需求状态

- 工具名：`planning-analytics-tracking`
- 存放位置：`.cursor/skills/planning-analytics-tracking/`
- 当前状态：已创建 Skill 与参考文档，待真实飞书策划案验证
- 创建日期：2026-05-11

## 目标

建立一个可复用 Cursor Skill，用于读取用户提供的飞书策划案链接，提取「详细方案」或其他功能相关章节，根据功能开发内容整理数据打点需求，并写回同一文档。

## 已确认规则

- 输入主要为飞书云文档链接，包含 `/docx/`、`/doc/`、`/wiki/`。
- 优先读取「详细方案」，找不到时读取功能说明、交互说明、流程、状态、规则、结算、奖励、异常等相关章节。
- 输出必须包含：参考表名格式（表名）、功能描述、字段明细表、打点说明总结。
- 字段明细表固定列为：通用表头、字段类型、字段说明、必要性、字段备注；如写入飞书表格，可在字段说明与必要性之间保留一个空列作视觉间隔。
- 负责策划只在页签顶部写一次，不在每个字段行重复。
- “通用表头”在本需求中表示字段名，不表示跨事件公共维度。
- 公共维度（如 player_id/user_id、server_id/zone_id、event_time、platform、app_version、session_id）单独成块说明，不在每张功能表重复。
- 表名默认使用全小写英文或团队约定缩写，一般不使用下划线；推荐结构为 `{业务域或玩法}{对象或模块}{生命周期动作}`，例如 `raidbattleend`。
- 用户确认优先尝试飞书原生底部页签；若 CLI/OpenAPI 不支持，再回退为文档内 `## 数据打点` 区块。
- 字段类型只支持 `string`；数值、枚举、百分比、布尔、时间和复合结构都在字段说明或字段备注中写清格式，不输出 `int`、`float`、`bool` 或 `json`。
- 最终交付优先使用“表名行 + 功能描述行 + 字段小表”的纵向块结构，不再使用所有字段平铺在一张横向大表的格式。
- 最终格式最前面必须增加 `需求描述总览`，按 `功能名 / 需求描述 / 目的` 汇总本次打点要回答的指标问题；示例只作为格式参考，实际内容必须从当前文档和事件表提炼，不能照搬。行数不固定，按实际指标完整提炼，不能因为参考示例是 3 行就只写 3 行。
- `美术需求`、`美术`、`视觉需求`、`表现需求`、`资源需求`、`动效需求`、`音频需求`、`素材需求` 等页签/章节默认进入读取黑名单，不作为功能实现口径生成字段。
- 需求稿只保留策划侧打点需求结构，不新增数据组后续方案字段；事件表不输出 `记录时机`、`打点侧建议`、`关联说明`，除非用户明确要求。
- 详细方案中的图片、长图、界面示意和流程图若承载核心交互/状态/流程，必须先读图再生成打点；无法稳定读取时先二次询问，不得只按文字补脑。

## 页签能力调研

- 已执行 `lark-cli docs --help`。
- 当前 CLI 仅提供 `+create`、`+fetch`、`+media-download`、`+media-insert`、`+search`、`+update`、`+whiteboard-update`。
- `lark-cli docx --help` 不存在对应命令。
- 搜索到的公开 OpenAPI 结果主要为 docx block 创建/读取接口，未直接确认存在原生 Tab 创建接口。
- 因此 Skill 中规定：执行真实文档时先调研原生页签能力；不能明确支持时，说明限制并回退为 `## 数据打点` 章节。

## 风险记录

- 如果策划案核心内容在图片、长图或画板中，必须先下载素材并进行人工/多模态确认；无法稳定识别时标记 `待确认`。
- 不得根据不完整截图或标题主观补全功能。
- 不得在未确认时覆盖原文档全文。
- 不要机械拆分过多事件表；如果展示、点击、未开放结果服务同一个复盘目标，可以合并到同一表。
- 对留坑、七日不做、未开放功能，只保留必要点击结果与报错字段，不设计完整功能闭环。

## 真实样例纠偏记录

- 日期：2026-05-11
- 样例：飞书电子表格 `基础-玩家个人信息` 的 `数据打点` 页签。
- 问题：初版输出使用横向大表，包含 `字段归属` 和逐字段 `负责策划`，同时把公共维度与功能字段混在一起，阅读负担过高。
- 用户修订后的方向：页签顶部写说明和负责策划；每张事件表用独立块呈现；字段列为通用表头、字段类型、字段说明、必要性、字段备注；公共维度单独成块；枚举和版本限制写入字段备注。
- 已同步：`.cursor/skills/planning-analytics-tracking/SKILL.md` 与 `reference.md`。
