# Beagle 专有名词术语表

## 使用说明
- 本表用于统一专有名词定义，避免规则、工具说明与需求文档中的口径不一致。
- 新增术语时，优先补充定义、使用场景和关联流程；若影响规则或工具流程，需同步更新 `rules/rules.md` 与 `session/`。

## 术语表
| 术语 | 定义 | 使用场景 | 关联流程 |
| --- | --- | --- | --- |
| 地块结构 | 用于描述地图、区域或模块格局的结构化表达。 | 玩法规则说明、地图相关 UI、数值与状态展示。 | 需求编写、术语统一、文案上下文说明 |
| 社交类型 | 用于区分好友、组队、工会、陌生人等社交关系类型的分类字段。 | 社交玩法、入口筛选、权限与文案差异描述。 | 页面拆分、文本提取、规则判断 |
| 友军判定 | 用于识别对象是否属于己方阵营或友方关系的规则。 | 战斗、协作、提示语与状态文案定义。 | 交互说明、提示文本生成 |
| 事件广播 | 模块间同步状态、事件通知或界面刷新所依赖的广播机制。 | 页面切换、弹窗响应、状态刷新、埋点或触发链说明。 | 交互路径分析、备注说明 |
| UX章节 | 按页面、弹窗、状态切换等粒度切分的原型结构单元。 | UX 长图拆分、章节截图、需求分段。 | `ux-text-requirements-generator` 页面切分流程 |
| 文本键值 | 用于管理 UI 文案标识、内容和备注的结构化记录。 | Excel 需求表、key 命名规则、文本提需。 | 文本需求生成、校验与导出 |

## 维护约定
- 若同一术语在不同工具中含义不同，需在定义中明确适用范围。
- 若术语影响到字段名、命名规范或产出模板，必须在相关需求级会话中记录变更。
