# 错题与复盘说明

## 目的
- 记录执行过程中出现的错误、误判和返工原因。
- 将一次性错误沉淀为后续任务可复用的防呆清单。

## 使用规则
1. 开始新任务前，先检索与当前任务最相关的错误分类。
2. 发现错误后，优先记录事实，再记录判断与修复。
3. 同类型错误尽量合并归纳，避免重复散落。

## 推荐分类
- `environment`：环境、依赖、脚本运行问题
- `rules`：规则遗漏、未同步、误解规范
- `ux-text`：截图切分、文本提取、key 规则误判
- `session`：会话文档遗漏、变更记录不完整

## 记录方式
- 单次事件可直接复制 `mistake-template.md` 形成新记录。
- 若某类错误高频出现，应额外整理成分类清单，放在本目录下长期维护。
