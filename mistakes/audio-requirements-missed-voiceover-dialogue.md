# 音频需求遗漏配音需求

## 基本信息
- 日期：2026-05-13
- 分类：rules
- 任务：根据飞书策划案生成音频需求

## 问题现象
- 音频需求 Skill 只覆盖音乐需求与音效需求，没有处理方案中出现对话、台词或角色提示时的配音需求。
- 如果策划案存在机器人提示、NPC 对话、角色对白、新手引导语音等内容，旧规则不会单独罗列说话人、情绪和触发场景。

## 原因分析
- 初版音频需求结构只参考了音乐与音效两个 part，未把“对话文本需要配音”作为独立判断条件。
- 对“文本/提示/机器人提示”只按界面文案理解，没有进一步判断是否存在说话人和语音表达需求。

## 修复动作
- 更新 `planning-audio-requirements` Skill：当方案出现对话、台词、角色语音、NPC 提示、机器人提示等内容时，单独输出 `配音需求`。
- 配音需求必须列出对话内容、说话人、情绪、场景触发。
- 未出现对话需求时，不为了凑格式输出空的配音表。

## 预防办法
- 阅读策划案时检索 `对话`、`台词`、`语音`、`旁白`、`说话人`、`机器人提示`、`NPC`、`文本`、`提示` 等关键词。
- 凡是有明确说话人或拟人化提示，都要判断是否进入配音需求。
- 普通 UI 文案、按钮、标题、说明、错误提示、IM 邀请、组队申请、系统通知等气泡窗文案，不默认作为配音需求；只有文档明确写明语音播报、角色/系统发声或明确说话人时才纳入。
- 不改写或新增台词，只罗列文档已有文案；说话人或情绪不明确时标 `待策划确认`。

## 执行前检查项
- [ ] 是否检查了方案中的对话、台词、机器人提示、NPC 提示
- [ ] 是否排除了仅作为 UI 展示的气泡窗文案、按钮文案和系统通知文案
- [ ] 是否单独输出配音需求 part
- [ ] 是否列明对话内容、说话人、情绪、场景触发
- [ ] 是否避免凭空创作新台词
