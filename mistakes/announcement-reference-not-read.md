# 公告需求未强制读取 reference

## 事件
- 日期：2026-05-21
- 任务：根据 ROK 策划案生成公告需求，并根据版本公告内容表修正公告需求 Skill

## 问题
- 执行公告需求 Skill 时，虽然主文件提到“按更新规模选择文案形态，详细判断与示例见 reference.md”，但前置流程没有强制读取 `reference.md`。
- 结果是实际生成时容易只依赖 `SKILL.md` 主规则，遗漏 `reference.md` 中的 ROK 润色列沉淀、输出模板和历史文案口径。

## 影响
- 公告文案可能过度展开成规则说明页，偏离版本公告短版口径。
- 可能遗漏历史文案中更重要的表达习惯，例如替代旧活动、先锋体验范围、国服/国际服差异、修正不影响实际效果等口径。

## 修复动作
- 在 `planning-announcement-requirements/SKILL.md` 的必读前置中加入本目录 `reference.md`，并明确“必须实际读取，不得只读本 Skill 主文件”。
- 在总体流程和完成检查中加入对 `reference.md` 的使用要求。
- 扩充 `reference.md` 的 ROK 润色列读取口径、常用句式、规则限制和活动上线写法。
