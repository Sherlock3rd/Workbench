# 数据打点页签写入漏格式

## 基本信息
- 日期：2026-05-26
- 分类：rules
- 任务：根据飞书电子表格策划案生成并写入数据打点页签

## 问题现象
- 创建新的 `数据打点` 页签后，只写入了内容，没有同步设置标题、分区、表头、边框、合并、列宽和行高。
- 初次补格式时使用固定行号，后半段合并范围偏移，导致部分 `表名`、`功能描述` 右侧内容被合并区域吞掉。
- 仅用 `sheets +read` 验证文字存在，误判为交付完成；用户在飞书页面看到的是未格式化或错位的表格。

## 原因分析
- 把“内容写入成功”误当成“飞书交付完成”，没有把页面可读性作为验收终点。
- 没有严格执行 `spec/feishu-document-writing-special-flow-spec.md` 中的写入后格式与导出复查流程。
- 没有优先复用或复刻已有数据打点页签的视觉模板，导致新页签默认裸写。
- 格式化时没有从最终行数据动态计算分区行、表头行和合并行，而是手写固定行号，容易在内容调整后错位。

## 修复动作
- 重建 `数据打点` 页签，重新写入精简版打点需求。
- 补充总标题、分区标题、表头、字段标签、全表边框、合并单元格、列宽和行高。
- 导出 xlsx 后用 `openpyxl` 复查标题底色/字色、合并范围、列宽、关键行高，并确认后半段 `表名`、`功能描述` 未被合并吞掉。
- 将本问题补充到 `planning-analytics-tracking` Skill 的飞书电子表格写回和验收规则中。

## 预防办法
- 只要写入飞书电子表格，完成定义必须包含：内容写入、格式应用、内容读取验证、元数据合并验证、导出 xlsx 视觉复查。
- 新建页签不能只写内容；必须同步设置标题/分区/表头/边框/列宽/行高，或明确说明当前仅为草稿未格式化。
- 合并范围、样式范围和行高必须基于最终 rows 数据动态生成，不能使用上一版或草稿阶段的固定行号。
- `sheets +read` 只能证明单元格值存在，不能证明格式正确；不得作为唯一验收依据。

## 执行前检查项
- [ ] 是否已先确认写入的是新页签、替换页签还是追加内容
- [ ] 是否已按最终 rows 动态计算分区行、表头行、字段块行和合并范围
- [ ] 是否已设置标题、分区、表头、边框、列宽和行高
- [ ] 是否已用 `sheets +read` 验证内容完整
- [ ] 是否已用 `sheets +info` 或导出文件验证合并范围没有错位
- [ ] 是否已导出 xlsx 并检查颜色、字号、字色、列宽、行高和关键合并区域
