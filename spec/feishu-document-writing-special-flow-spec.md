# 飞书文档写入特别流程规范

## 目标

- 固化本项目中飞书文档、飞书电子表格写入时踩过的格式问题和修复流程。
- 为所有会写回飞书文档或表格的 Skill 提供统一前置规范。
- 避免“接口返回成功但用户页面看起来缺文本、错位、行高错误、颜色错误”的问题再次发生。

## 适用场景

所有 Skill 或脚本只要需要写入、追加、替换、创建或格式化飞书内容，都必须先读取本规范，包括但不限于：

- 使用 `docs +update` 写入飞书云文档章节。
- 使用 `sheets +write` 写入飞书电子表格单元格。
- 使用 OpenAPI 创建页签、合并单元格、设置样式、设置行高列宽。
- 从参考页签复制或复刻样式。
- 对已有飞书文档/表格做二次修复。

## 写入前确认

- 写入或修改飞书文档/表格前，必须确认用户意图；若用户已明确要求“生成到这个文档/页签/表格”，可继续。
- 不覆盖原策划案正文；优先新增页签、追加章节或替换用户明确指定的区块。
- 如果已有同名页签或区块，先判断是复用、替换、另建还是保留旧版。不能静默覆盖有价值内容。
- 对格式、字段命名、标题文案等会影响模板规范的修改，要同步写入对应 Skill 和 `session/session.md`。

## 写入实现规则

### 1. Windows 与 JSON 写入

- PowerShell 下长 JSON 和中文内容容易因为命令行长度、转义或编码出错。
- 复杂写入优先用 Python 构造 JSON，再通过 `subprocess` 调用 `lark-cli.cmd`。
- JSON 使用 `json.dumps(..., ensure_ascii=False)`，避免中文被破坏。
- Windows 下优先调用 `lark-cli.cmd`，不要假设 `lark-cli` 可直接被 Python 找到。
- 大范围写入不要一次把超长 JSON 作为命令参数传入，容易出现“命令太长”或写入失败；应按小批次分段写入。

### 2. 电子表格结构

- 先 `sheets +info` 获取页签 ID、页签名称、合并信息和表格元数据。
- 写入前记录目标页签 ID；用户给的是 URL 中 `sheet=<id>`，不要误写到默认页签。
- 如果创建新页签，创建后再次读取 `sheets +info` 确认真实 `sheet_id`。
- 合并单元格后，内容只应写在合并区域左上角单元格；不要把内容写到合并区域中间或尾部。
- 读取合并单元格时，非左上角常返回 `null`，不能因此判断内容缺失。

### 3. 行号与动态布局

- 不要把上一版生成结果的固定行号硬套到新内容。音乐条目、音效条目、待确认事项数量变化后，所有分区行、条目标题行、字段行、空白行都必须重新计算。
- 写样式时必须使用最终实际行号，而不是草稿阶段的预计行号。
- 行高、合并范围、标题样式、字段标签样式都要从同一份最终行号表生成。
- 如果后续补写、插入、删除了某一行内容，必须重新检查后续所有硬编码行号是否已经偏移。

### 4. 样式设置顺序

推荐顺序：

1. 写入内容。
2. 合并单元格。
3. 设置列宽。
4. 设置全表基础字体和基础对齐。
5. 单独恢复总标题、分区标题、条目标题、字段标签、正文等局部样式。
6. 设置行高。
7. 导出或读取验证。

注意：

- 全表基础样式会覆盖局部样式，所以局部样式必须在全局样式之后再次设置。
- 如果再次全表重设字体/对齐，必须重新恢复大标题字号、深色底白字、条目标题底色。

## 常见问题与防呆

### 1. 文本看起来缺失

现象：

- `sheets +read` 能读到完整文本，但飞书页面上只显示前半段。
- 合并单元格里长文本被裁切，看起来像后半段没写入。

原因：

- 飞书样式接口未稳定暴露自动换行能力。
- 合并单元格不会可靠自动撑高。
- 行高太低，长文本被视觉截断。

处理：

- 对长文本主动插入 `\n`，尤其是 `说明：`、`一句话解释：`、`关键交互：`、`前置条件：`、`待确认事项`。
- 给包含长文本的最终实际行设置足够行高。
- 写回后读取关键单元格确认值完整，再导出 xlsx 检查行高。

### 2. 行高错误

现象：

- 长文本行过矮。
- 空白行过高。
- 标题行和正文行高度不一致。

原因：

- 行高设置用错行号。
- API 行高索引与肉眼行号混淆。
- 内容行变化后没有重算行高。

处理：

- 飞书开放平台的 `dimension_range` 只提供 `fixedSize` 设置行高/列宽，没有确认可用的 UI 版真正 `autofit row height` 接口；完成写入后应执行“模拟 autofit”：读取最终所有行内容，按显式换行数、文本长度、行类型为每一行计算行高，再批量写回 `fixedSize`。
- 模拟 autofit 时，已手动插入 `\n` 的文本优先按显式换行数计算，不要再按字符长度重复放大，否则两行文本会被估成三四行。
- 根据最终内容行号设置行高，不使用上一版固定行号。
- 长文本行设置为明显高于普通行的高度。
- 空白行保持紧凑。
- 调用 `dimension_range` 后必须导出 xlsx 复查真实行高；不要只相信 API 成功返回。若发现目标行被相邻空白行设置覆盖，立即按导出里的实际行号补正。
- 导出 xlsx 后用 `openpyxl` 读取 `row_dimensions[row].height` 核对关键行。

### 3. 标题行错位

现象：

- 序号在某一行，但标题样式或合并区域在下一行。
- 例如 `C46` 是序号 `3`，但 `D46:H46` 没有标题样式，反而 `D47:H47` 被合并或着色。

原因：

- 条目标题行由旧模板硬编码。
- 插入/清空行后，只改了内容，没改合并和样式范围。

处理：

- 条目标题必须满足：序号格和标题格在同一行。
- 检查 `C<row>` 是序号时，`D<row>:H<row>` 是否为对应标题合并区域。
- 清理误套样式的空白行，避免空白行带浅黄色或标题合并。

### 4. 字段标签列错位

现象：

- `说明：`、`一句话解释：`、`关键交互：` 等字段标签出现在 `B` 列。
- `C` 列为空，正文在 `D` 列，视觉上整体偏左/偏乱。

正确规则：

- `B` 列只用于一级分区序号。
- `C` 列用于条目序号和字段标签。
- `D` 列开始放标题正文或字段正文。
- 字段标签必须在 `C` 列，正文从 `D` 列开始。

### 5. 颜色和字号被覆盖

现象：

- 深灰底单元格变成黑字。
- 大标题字号被压成正文大小。
- 黄色层级不一致。

正确规则：

- 主黄色：`#FAC603`。
- 浅黄色：`#FAF1D1`。
- 深灰色：`#3A3E43`。
- 深灰底必须白字：`#FFFFFF`。
- 总标题用 Calibri 36 加粗；全表批量设置后必须单独恢复。

### 6. 合并信息导致误判

现象：

- 读取范围时，合并区域非左上角显示 `null`。
- 用户看到页面空，Agent 误以为没写入。

处理：

- 合并区域只以左上角单元格为准。
- 用 `sheets +info` 检查 `merges`，确认合并范围是否覆盖了正确行列。
- 需要核对视觉效果时，导出 xlsx 后检查 `merged_cells.ranges`。

### 7. 接口成功不等于页面正确

现象：

- OpenAPI 返回 `success`，但页面上仍错位、截断、颜色不对。

处理：

- 每次重要写入后至少做两类验证：
  - 内容验证：`sheets +read` 读取关键范围，确认文本值、换行、列位置。
  - 视觉验证：导出 xlsx，用 `openpyxl` 检查行高、合并范围、字号、底色、字色。
- 如果用户指出格式问题，不要只重复读值；必须检查合并、行高、样式和导出结果。

## 推荐验证清单

写入飞书电子表格后，至少检查：

- `A1` 总标题是否存在、字号是否为大标题。
- 分区行：`B` 列为序号，`C:H` 为分区标题。
- 条目标题行：`C` 列为条目序号，`D:H` 为条目标题。
- 字段行：`C` 列为字段标签，`D:H` 为正文。
- 深灰底 `#3A3E43` 的字体是否为白色 `#FFFFFF`。
- 长文本是否包含必要换行，行高是否足够。
- 空白行是否没有误套标题底色或合并。
- 待确认事项等尾部区域是否没有行号偏移。
- 读取范围没有重复文本、错位文本或旧文本残留。

## 推荐导出复查方式

当涉及复杂表格样式、合并、行高时，写回后导出 xlsx：

```powershell
lark-cli.cmd sheets +export --url "<sheet_url>" --file-extension xlsx --output-path ".cursor\tmp\<name>.xlsx" --as user
```

再用 `openpyxl` 检查：

```powershell
py -3 -c "from openpyxl import load_workbook; ws=load_workbook(r'.cursor\tmp\<name>.xlsx', read_only=False, data_only=True)['<sheet_name>']; print(ws.row_dimensions[46].height); print(list(ws.merged_cells.ranges)[:20])"
```

复查完成后删除临时导出文件和临时脚本，避免占用空间。
